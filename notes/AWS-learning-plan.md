# AWS Learning Plan (SAA-level)

## Theme
Evaluating whether and how to learn AWS (Solutions Architect Associate level)

## Objective
- Understand Cloud / GSI architecture concepts
- Improve ability to communicate in cloud-native language
- Strengthen platform and architecture thinking

## Background
- Target roles: DevRel / Platform / Ecosystem / Architect
- Lack of Cloud / GSI experience can be a gap
- Existing strengths: API / Docker / JSON

## Approach
Focus is not on certification, but on:
→ Understanding architecture  
→ Being able to explain it clearly  

## Scope
- AWS fundamentals (EC2, S3, VPC)
- High availability design
- Scalable architecture
- Security (IAM basics)

## Timeline
1–2 months (focused learning)

## Expected Outcomes
- Ability to discuss cloud / GSI architecture
- Stronger explanation of structural similarities across systems
- Better alignment with platform and architect roles

## Key Principle
This is not about adding a new skill,
but about improving translation ability between domains.

## Why AWS
AWS is used as a reference to understand modern cloud architecture patterns that are widely adopted across platforms and ecosystems.
