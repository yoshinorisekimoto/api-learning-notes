# Step 7 – Review and Articulation

## What I learned
- I consolidated my understanding of API fundamentals by explaining the concepts in a structured way
- I focused on being able to clearly articulate how APIs work as systems, not just individual endpoints

## Summary
I learned the fundamentals of APIs through hands-on practice using Postman.

APIs are interfaces that allow systems to communicate using standardized request and response formats, typically JSON.

REST APIs use HTTP methods such as GET, POST, PUT, and DELETE to interact with resources.

Requests are structured using path parameters, query parameters, and headers.

JSON represents structured data and serves as the data model for APIs.

Authentication is handled through headers using API keys or OAuth tokens.

Collections group related APIs, and environment variables enable reuse and scalability across different environments.

## Why it matters
Being able to clearly explain APIs demonstrates not only technical understanding but also the ability to think in terms of system design and architecture.

## Key Insight
APIs should be understood as part of a system architecture, not just individual requests.
